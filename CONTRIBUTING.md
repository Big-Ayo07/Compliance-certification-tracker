# Contributing Guidelines

Thank you for your interest in contributing to the Compliance Certificate Tracker!

## Issues & Improvements

To contribute:

1. **Report Issues**: If you find bugs or have suggestions, please open an issue on GitHub
2. **Documentation**: Help improve documentation and examples
3. **Code Improvements**: Submit pull requests with enhancements

## Development

### Local Setup

1. Clone the repository
2. Open `compliance certificate tracker/index.html` in a web browser
3. Make your changes
4. Test thoroughly before submitting

### Testing

- Test all browser functionality
- Verify spreadsheet import/export works correctly
- Check responsive design on mobile devices
- Validate date calculations

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing naming conventions
- Keep functions focused and well-documented
- Comment complex logic

## Submitting Changes

1. Create a descriptive commit message
2. Reference any related issues
3. Provide clear explanation of changes
4. Ensure all tests pass

Thank you for contributing!
